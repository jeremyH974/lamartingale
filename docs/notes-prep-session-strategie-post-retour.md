## Comment lire le retour Stefani

### Signaux à chercher activement

**Signal 1 — Réaction émotionnelle au pack**
- "C'est exactement ce que je cherchais" / "Bluffant" → signal très positif
- "Intéressant" / "Je vais regarder en détail" → signal moyen, à creuser
- "Pas mal" / "OK" → signal faible, à creuser fort
- Silence sur le contenu, focus sur autre chose → signal mitigé

**Signal 2 — Vocabulaire spontané sur ce qui pourrait manquer**
- "J'aimerais pouvoir [verbe d'action]" → c'est une demande active. Note-la.
- "Et si on ajoutait [feature]" → idem
- "Comment ça marche pour [cas d'usage]" → curiosité, pas demande encore

**Signal 3 — Réaction au PS roadmap V2** (si inclus dans email)
- Stefani mentionne **explicitement** une des 3 directions (Studio / Brief interactif / Dashboard rétention) → priorité validée
- Stefani mentionne une **autre** direction non-PS → signal très précieux : il a un besoin que tu n'avais pas anticipé
- Stefani **ne mentionne pas** le PS → soit il n'a pas vu, soit pas prioritaire pour lui (à creuser)

**Signal 4 — Discussion business**
- "Combien ça coûte" / "Comment on travaillerait ensemble" → signal d'intérêt commercial fort
- Pas de discussion business → soit prématuré, soit pas convaincu par la valeur

**Signal 5 — Mention de Christofer**
- "J'en parle à Christofer" → potentielle implication business/finance
- "Christofer trouve que..." → signal qu'il y a déjà eu discussion interne
- Christofer répond directement → signal d'intérêt structuré

### Pièges à éviter dans la lecture

**Piège 1 — Sur-interpréter un signal positif**
"Bluffant" peut être de la politesse. Ne pas conclure à une signature imminente sur un compliment isolé. Attendre le 2e signal concret (demande de détail, demande de rendez-vous, question pricing).

**Piège 2 — Sous-interpréter un signal critique**
"Le ton ne nous parle pas" est PRÉCIEUX. C'est exploitable. C'est mille fois plus utile qu'un "intéressant" évasif. Ne pas se vexer, creuser.

**Piège 3 — Confondre silence et refus**
14 jours de silence n'est pas un refus. C'est probablement un signal "très chargé / pas de temps / le pilote n'est pas en haut de la pile". Une relance courte et factuelle est légitime.

**Piège 4 — Réagir à chaud**
Quel que soit le retour, **attendre 24h avant de répondre**. Discuter avec quelqu'un de confiance entre temps. Le marathon a accumulé de la fatigue émotionnelle, les premières réactions ne sont pas toujours les meilleures.

## Les 4 scénarios de retour + plan d'action dans chaque cas

### Scénario 1 — Retour positif structuré

**Symptômes** :
- Compliments sur la qualité éditoriale
- Mention spontanée d'un cas d'usage où Stefani veut utiliser Sillon
- Demande de rendez-vous pour discuter suite
- Possible mention business

**Plan d'action immédiat (semaine du retour)** :

1. **Répondre dans les 24-48h** : remerciement, demande de RDV pour structurer la suite (visio 30-45 min)

2. **Préparer le RDV** :
   - Lire `strategy-ideas-backlog.md` Catégorie B (conditionnels)
   - Identifier laquelle des B1-B4 correspond au signal Stefani
   - Préparer 2-3 questions ouvertes pour creuser le besoin réel
   - **NE PAS proposer de roadmap fixe** — écouter d'abord, proposer ensuite

3. **Pendant le RDV** :
   - Écouter 70% / parler 30%
   - Creuser ce que Stefani veut faire avec Sillon, pas ce qu'il "trouve cool"
   - Ne pas trancher pricing/business model en RDV — dire "je te reviens avec une proposition structurée"

4. **Post-RDV (semaine suivante)** :
   - Activer la session "Sillon stratégie post-retour" claude.ai avec les éléments du RDV
   - Décider quelle direction Catégorie B activer
   - Mettre à jour `strategy-ideas-backlog.md` (B → A)
   - Rédiger un brief Phase 8 ciblé sur la direction validée

**Ce qu'il NE faut PAS faire** :
- Promettre une livraison V2 sans avoir scopé
- Discuter pricing en premier RDV (sauf si Stefani le demande explicitement)
- Sur-vendre les fonctionnalités V2 alors qu'elles ne sont pas construites

### Scénario 2 — Retour positif partiel

**Symptômes** :
- Compliments sur certains livrables, critiques sur d'autres
- "Le ton est bon mais [problème spécifique]"
- "Les cross-refs sont géniales mais [autre problème]"
- Pas de demande de RDV spontanée

**Plan d'action** :

1. **Répondre dans les 48-72h** : remerciement détaillé, **acknowledger les critiques** (ne pas se défendre), poser 2-3 questions précises pour creuser

2. **Analyser le retour** :
   - Lister les points positifs (à préserver impérativement)
   - Lister les points critiques (à fixer en priorité)
   - Identifier si les critiques sont fixables avec Phase 8 (Catégorie A) ou nécessitent V2 (Catégorie B)

3. **Décider** :
   - Si critiques fixables Phase 8 (ex: "le ton manque de mordant" → calibrer prompts) : démarrer Phase 8 avec scope ciblé
   - Si critiques V2 (ex: "j'aimerais éditer interactivement") : proposer un RDV pour spec ensemble

4. **Communication structurée** :
   - "Voici comment je propose de répondre à tes points : [plan d'action 2-3 semaines]. Tu veux qu'on en parle avant que je code ?"

**Ce qu'il NE faut PAS faire** :
- Ignorer les points négatifs et focus sur les positifs
- Tout fixer en même temps (overload)
- Trancher unilatéralement sans valider avec Stefani

### Scénario 3 — Retour négatif ou tiède

**Symptômes** :
- "Pas convaincu" / "Pas pour nous" / "Pas le bon moment"
- Critique structurelle (pas juste détails)
- Pas de demande de RDV ni d'ouverture sur la suite
- Possible silence après réception

**Plan d'action** :

1. **Premier mouvement : ne pas argumenter à chaud**
   - Répondre simplement : "Merci pour le retour franc. J'aimerais comprendre [point précis] : pourrais-tu me dire pourquoi [...] ?"
   - Pas de défense, pas de "mais regarde aussi..."

2. **Analyser à froid (24-48h)** :
   - Le retour est-il sur la qualité ? Le timing ? Le pricing ? Le scope ?
   - Stefani a-t-il mentionné quelque chose qui pourrait te servir avec d'autres prospects ?
   - Y a-t-il des éléments factuels à corriger dans le pack (erreurs ?) ?

3. **Décisions stratégiques** :
   - **Si le retour pointe un problème de qualité** : lecture critique sévère du pack, identification du gap, plan de remédiation
   - **Si le retour pointe un problème de fit** : comprendre pourquoi Sillon ne matche pas Orso. Est-ce généralisable à d'autres prospects ?
   - **Si le retour pointe un problème de timing** : "rappel-toi dans 6 mois" est légitime, classer comme "warm lead" sans réponse immédiate

4. **Pivot ou persévérance** :
   - **Pivot** : si retour identifie un problème structurel du produit/marché. Pas honteux. Sillon peut être un bon produit mais avec un autre client cible que Stefani.
   - **Persévérance** : si retour est un "non" personnel à Stefani mais le produit reste défendable, démarcher d'autres podcasteurs (Hugo Décrypte, Bababam, Choses à Savoir, etc.) avec le pack adapté

**Ce qu'il NE faut PAS faire** :
- Insister auprès de Stefani malgré le non
- Conclure que le projet est mort sur un seul retour
- Re-démarcher Stefani 1 semaine plus tard avec les "améliorations"

### Scénario 4 — Silence absolu (>14 jours)

**Symptômes** :
- Pas de réponse à l'email pilote
- Pas de réception confirmée non plus
- Aucun signal après 14 jours

**Plan d'action** :

1. **Vérifier la délivrabilité** :
   - Email arrivé (pas spam, pas filtre) ?
   - Lien Drive accessible si utilisé ?
   - Test envoi à un compte test pour confirmer

2. **Relance courte (J+14 à J+18)** :
   ```
   Hey Matthieu,
   
   Je voulais juste vérifier que le pack est bien arrivé. 
   Je sais que tu es très chargé — pas de pression sur le retour, 
   juste t'indiquer que je suis dispo si tu as 15 min un jour.
   
   Cheers,
   Jérémy
   ```

3. **Si toujours silence après J+30** :
   - Considérer ce silence comme un "non poli"
   - Pas de relance supplémentaire
   - Activer démarchage autres prospects (cf. liste podcasteurs FR)
   - Garder Sillon prêt pour le jour où Stefani revient (3 mois ? 6 mois ? un an ?)

4. **Ne pas se décourager** :
   - Le silence n'est pas un jugement de qualité
   - Beaucoup de prospects "warm" reviennent 6-12 mois plus tard
   - Le travail réalisé pour Stefani sert de pitch package pour les prochains

**Ce qu'il NE faut PAS faire** :
- Plus de 1 relance
- Le harceler sur LinkedIn / Twitter
- Re-pitcher avec un email plus long
- Considérer le projet comme mort
